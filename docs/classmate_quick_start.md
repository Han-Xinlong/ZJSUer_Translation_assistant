# 给同学体验前端交互的快速指南

这份说明写给没有接触过开发、但想从 GitHub 把项目跑起来体验页面交互的同学。

项目地址：

```text
https://github.com/Han-Xinlong/ZJSUer_Translation_assistant
```

## 你能体验到什么

本项目是一个面向外语学习者的“随写随翻”AI 工具原型。按下面步骤启动后，你可以在浏览器里体验：

- 写作台：输入中文、日文、英文等文本。
- 随写随翻：选择快速模式或深度模式，查看翻译结果。
- 随写随修：让系统对表达进行润色和解释。
- 表达库：收藏值得积累的好表达。
- 表达改进库：记录值得优化的句子和修改建议。
- 历史详情：回看每一次翻译、润色记录。
- 学习档案：查看今日目标、累计表达、连续学习和趋势图。
- 社群互学：模拟把优秀表达分享到学习社区。

如果只是想马上体验线上版本，可以访问国内体验地址：

```text
http://62.234.13.61/
```

当前线上版本已接入 DeepSeek 真实模型。本地从 GitHub 跑项目时，后端默认使用 mock 模式，所以不需要配置真实 AI Key，也能完整体验产品流程。

## 第一步：准备环境

请先确认电脑里安装了下面两个工具：

- Node.js：用于启动前端页面。
- Python 3：用于启动后端服务。

如果你不确定是否已经安装，可以打开终端，分别输入：

```bash
node -v
python3 --version
```

只要能看到版本号，通常就可以继续下一步。

## 第二步：下载项目

打开终端，进入你想存放项目的目录，然后执行：

```bash
git clone https://github.com/Han-Xinlong/ZJSUer_Translation_assistant.git
cd ZJSUer_Translation_assistant
```

如果你不会使用 Git，也可以在 GitHub 页面点击绿色的 `Code` 按钮，选择 `Download ZIP` 下载压缩包，然后解压到本地。

## 第三步：启动后端服务

后端负责处理翻译、润色等请求。请在项目根目录执行：

```bash
bash scripts/dev_backend.sh
```

第一次启动会自动安装 Python 依赖，可能需要等待一会儿。看到类似下面的内容，说明后端启动成功：

```text
Uvicorn running on http://127.0.0.1:8000
```

这个终端窗口不要关闭，让它保持运行。

## 第四步：启动前端页面

重新打开一个新的终端窗口，进入同一个项目目录，然后执行：

```bash
bash scripts/dev_frontend.sh
```

第一次启动会自动安装前端依赖，也可能需要等待一会儿。看到类似下面的内容，说明前端启动成功：

```text
Local: http://localhost:5173/
```

然后打开浏览器访问：

```text
http://localhost:5173
```

现在就可以体验前端交互了。

## 如果只想看页面布局

如果你只是想简单看一下页面长什么样，也可以只启动前端：

```bash
cd frontend
npm install
npm run dev
```

然后访问：

```text
http://localhost:5173
```

不过这种方式没有启动后端，所以点击“翻译”“润色”等按钮时，请求可能会失败。想体验完整流程，建议按前面的步骤同时启动后端和前端。

## 常见问题

### 1. npm install 等很久没有反应怎么办？

第一次安装前端依赖时需要从网络下载包，速度取决于网络情况。可以先等待几分钟。

如果长时间没有变化，可以尝试：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

然后再运行：

```bash
npm run dev
```

### 2. 浏览器打不开 localhost:5173 怎么办？

先确认前端终端里是否显示了：

```text
Local: http://localhost:5173/
```

如果没有，说明前端还没有启动成功。请检查终端里是否有报错。

### 3. 翻译按钮点击后报错怎么办？

请确认后端服务也已经启动，并且后端终端里显示：

```text
http://127.0.0.1:8000
```

前端会把 `/api` 请求转发给后端。如果只启动了前端，没有启动后端，翻译和润色请求就无法完成。

### 4. 需要 AI API Key 吗？

本地体验不需要。当前项目后端默认是 mock 模式，会返回模拟的翻译、润色和审校结果，适合演示和课堂体验。

如果后续要接入真实 AI 模型，例如 DeepSeek、OpenAI 或 DashScope，可以参考 [backend/README.md](../backend/README.md)。

## 推荐体验路线

第一次体验时，可以按这个顺序操作：

1. 打开 `http://localhost:5173`。
2. 在左侧写作台输入一段中文或外语句子。
3. 选择目标语言，比如英文或日文。
4. 点击“快速翻译”，观察译文结果。
5. 再切换到深度模式，比较结果和审校建议。
6. 点击“润色”，查看表达修改说明。
7. 收藏一条表达，进入表达库查看。
8. 保存一条改进记录，进入表达改进库查看。
9. 打开历史详情，查看刚才的操作记录。
10. 打开学习档案，查看学习数据变化。

这样基本就能了解这个项目的主要交互闭环。
