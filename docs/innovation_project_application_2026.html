二、项目摘要

本项目面向外语学习者在写作与翻译训练中长期存在的“写不出、翻不透、改完记不住”问题，开发一款轻量化、可复盘、会督促的“随写随翻”AI学习工具。项目以单页Web应用为主要形态，围绕“智能输入—随写随翻—随写随修—学习沉淀—成长可视—社群互学”形成完整学习闭环，帮助学习者把一次性的AI结果转化为可复习、可迁移、可持续积累的语言学习资产。

与传统翻译软件只提供结果、专业CAT平台学习门槛较高不同，本项目强调“轻量陪伴，深度成长”。系统支持快速翻译、深度翻译、AI润色、语境说明、语音输入、沉浸写作、历史复盘、表达库、错题库、每日目标、学习热力图、学习报告导出、演示数据加载和本地社群分享等功能。后端采用FastAPI封装大模型调用，设计MockProvider、OpenAIProvider和OpenAI-compatible Provider，前端采用React + Vite构建学习工作台，并已通过Docker部署到腾讯云轻量服务器，形成可直接访问的国内演示版本。

截至2026年5月3日，项目已完成MVP原型、组件化重构、演示工具、学习报告导出、Vercel国际演示链路和腾讯云国内部署链路，并在腾讯云环境接入DeepSeek真实模型。当前版本已从mock演示进入真实AI小范围测试阶段，后续将继续开展用户测试、优化Prompt效果，并进一步形成技术文档、用户手册、研究报告、软著材料和竞赛展示材料。

三、立项依据与研究背景

3.1 现实背景：AI正在改变外语学习，但学习闭环仍不完整

生成式人工智能和大语言模型快速进入外语学习场景，翻译、润色、写作辅助、语法纠错等能力变得触手可及。然而，真实学习过程并不只是“获得一个译文”。外语学习者更需要在写作启动、语境判断、表达优化、错误复盘、长期积累和持续练习之间建立闭环。当前通用翻译工具往往重结果轻过程，学习者看完译文后很难知道为什么这样改；专业翻译平台功能强大但偏向职业译者，普通学生使用成本较高；词典和写作助手则常常缺少复盘、打卡和学习资产管理能力。

本项目从外语学习者的日常任务出发，将AI能力转化为“随写随翻、随写随修、随学随存”的学习伙伴。项目并非单纯追求机器翻译质量，而是关注AI介入后学习者能否真正理解、复盘和迁移表达。

3.2 竞赛与优秀项目启示

近年浙江省大学生创新创业相关赛事呈现明显趋势：优秀项目通常不只展示单点技术，而是将“AI+具体场景”落到真实需求、可运行产品、验证数据和推广路径上。以浙江省国际大学生创新大赛（2024）为例，公开新闻显示本届赛事汇聚全省113所高校、5.19万项创新项目，吸引39万名学生参与，并有563项进入省级决赛；项目内容覆盖新工科、新医科、新农科、新文科等多个领域。杭州师范大学、温州大学等高校获奖项目中也可看到“AIGC赋能教育”“AI数字人”“文化出海”“非遗传播”等方向，普遍强调技术与教育、文化、产业或公益场景的深度结合。

本项目吸收这些优秀项目的申报表达方式：一是把痛点写实，不停留在“AI很热门”；二是突出原型与部署成果，证明项目不是纸面设想；三是强调外语教育场景价值，体现新文科与人工智能融合；四是设计可验证的用户测试与学习效果评价路径，为后续结题和竞赛展示留出证据链。

3.3 项目价值

  • 学习价值：帮助外语学习者从AI译文和润色结果中提取表达、错误、修改理由和语境策略，减少“复制答案式”使用AI。
  • 教学价值：将AI修改过程转化为写作与翻译课程中的案例材料，支持教师开展过程性评价与课堂讨论。
  • 技术价值：探索轻量Web应用、大模型Prompt工作流、本地学习数据沉淀和国内外双线部署的完整实现路径。
  • 推广价值：项目门槛低、可在线访问、默认mock模式可无密钥演示，适合同学试用、答辩展示和后续校园推广。

五、主要研究内容与产品功能

5.1 智能输入模块

支持文本输入、语音录入、语境说明和沉浸式写作。学习者可以在课程作业、正式邮件、校园生活分享等不同场景下提供语境，使AI输出不再停留在机械直译，而是贴合实际表达目的。

5.2 随写随翻模块

提供快速模式和深度模式。快速模式用于短句即时查询；深度模式采用“初译+审校”的双阶段工作流，先生成初译，再从准确性、自然度、语境适配、表达价值等角度审校，输出最终译文和学习建议。

5.3 随写随修模块

面向学习者原文提供语法、词汇、风格、逻辑和表达自然度优化，并给出修改说明。学习者可将优秀表达加入表达库，将问题说明加入错题库。

5.4 学习复盘模块

系统保存每次翻译或润色历史，支持查看“原文—译文/终稿—建议/审校说明”的完整链路。历史详情可继续收藏表达和加入错题，形成复盘闭环。

5.5 成长可视与激励模块

提供学习档案、近7天练习趋势、14天热力格、每日目标、今日完成进度和连续学习天数，帮助学习者将零散练习转化为持续习惯。

5.6 演示工具与学习报告模块

新增演示数据加载、清空数据和学习报告导出功能。该模块特别适合答辩和同学试用：评审可快速看到完整闭环，学习者也可导出个人复盘材料。

5.7 语料推荐与社群互学模块

系统根据输入内容和AI输出推荐相关表达，学习者可将译文或润色结果分享到本地社群列表，形成语料共享与同伴启发的初步机制。

六、技术路线与系统架构

项目采用“前端轻量、后端服务化、模型可替换、数据本地化、部署双线化”的技术路线。前端使用React和Vite构建单页应用，后端使用FastAPI封装模型调用和AI工作流，数据优先保存在本地浏览器存储中，部署上同时保留Vercel国际演示链路和腾讯云国内稳定访问链路。

层级技术/模块说明
前端层React 18 + Vite实现写作台、结果展示、表达库、错题库、学习档案、历史详情、社群视图和演示工具。
组件层Sidebar、WorkspaceView、InsightPanel等已完成App.jsx组件化重构，降低后续维护成本。
可视化层ECharts动态加载实现学习趋势图和热力格，并减少首屏资源体积。
服务层FastAPI提供健康检查、状态查询、翻译、润色等接口。
AI编排层AIOrchestrator + Prompt模板封装快速翻译、深度初译、深度审校和润色工作流。
模型适配层MockProvider / OpenAIProvider / OpenAI-compatible Provider默认mock可演示，已支持DeepSeek真实模型，也可扩展OpenAI、DashScope、通义、文心等。
数据层localStorage，后续IndexedDB保存历史记录、表达库、错题库、目标配置和社群分享。
部署层Docker Compose + Nginx + 腾讯云国内地址通过Nginx统一暴露前端与后端，同源访问 `/api/...`，减少跨域和网络不稳定问题。

系统流程为：用户输入文本与语境 → 前端发起翻译/润色请求 → FastAPI接收请求 → AIOrchestrator加载Prompt并调用Provider → 返回结构化JSON → 前端展示译文、建议与修改说明 → 用户收藏、复盘、打卡、导出报告和共享。

七、项目创新点

  • 从“给答案”到“促学习”：项目不把AI翻译结果作为终点，而是将结果、审校说明、修改建议转化为可复盘的学习资产。
  • 双模式AI工作流：快速模式满足即时查询，深度模式采用初译+审校机制，兼顾效率和学习深度。
  • 本地化学习闭环：历史记录、表达库、错题库、热力图、每日目标和学习报告共同构成“练习—沉淀—复盘—再练习”的闭环。
  • 适合新文科场景：项目由外语学习痛点出发，将AI工程能力转化为语言学习工具，体现“人工智能+外语教育”的交叉创新。
  • 低门槛演示与部署：mock模式无密钥即可演示，腾讯云国内部署和DeepSeek真实模型接入已验证，便于答辩、课程试用和校内推广。
  • 可扩展AI架构：Provider抽象和Prompt模板分离，使后续接入不同大模型、增加评价模块和云端同步成为可能。

十三、风险分析与应对措施

风险表现应对措施
模型调用成本与稳定性真实API调用可能产生费用或受到网络影响。保留mock模式；设置调用额度；支持多Provider扩展；国内展示优先保证基础功能可用。

前端功能复杂度增加功能堆叠导致维护困难。持续组件化；增加工具函数和单元测试；保持功能边界清晰。
学习效果难量化项目成效不易证明。设计前后测、用户访谈、使用日志、翻译质量评分和学习报告样例。
国内访问与部署维护Docker Hub、GitHub、模型API等可能受网络影响。使用腾讯云镜像源、PyPI国内镜像、Docker Compose部署文档和本地可回滚方案。

十四、用户测试与评价方案

后续将选择外语专业学生和有外语写作需求的非外语专业学生开展小规模测试。评价指标包括：使用前后表达修改质量、学习者对修改理由的理解程度、表达库/错题库使用频率、每日目标完成情况、学习报告可读性和用户满意度。

  • 前测：收集学习者原始写作/翻译样本，记录常见错误类型。
  • 过程测试：观察学习者使用快速翻译、深度翻译、润色和收藏功能的路径。
  • 后测：对比修改质量、表达复用情况和错题复盘效果。
  • 访谈：收集用户对功能可用性、学习价值和界面体验的反馈。
