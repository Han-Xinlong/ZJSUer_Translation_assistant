# 使用手册：随写随翻 AI 学习工具

版本：MVP 初版 + 真实模型测试版
适用对象：外语学习者、项目团队、指导教师、答辩评审  
访问地址：本地开发环境 `http://127.0.0.1:5173/`；国内体验地址 `http://62.234.13.61/`

## 1. 产品简介

“随写随翻”是一款面向外语学习者的 AI 写作与翻译学习工具。它不是只给译文的一次性翻译器，而是一个可以帮助你写、翻、改、存、复盘和打卡的学习伙伴。

你可以用它完成：

- 注册/登录自己的学习账号。
- 写一段中文/外语文本。
- 快速翻译。
- 深度翻译并查看审校说明。
- 润色外语表达。
- 收藏优秀表达。
- 记录表达改进建议和修改说明。
- 查看历史详情。
- 设置每日目标。
- 查看学习趋势和热力图。
- 将好句分享到本地社群。

## 2. 启动方式

### 2.1 启动后端

在项目根目录运行：

```bash
bash scripts/dev_backend.sh
```

成功后后端地址为：

```text
http://127.0.0.1:8000/
```

### 2.2 启动前端

新开一个终端，在项目根目录运行：

```bash
bash scripts/dev_frontend.sh
```

成功后前端地址为：

```text
http://127.0.0.1:5173/
```

### 2.3 默认模式

系统默认使用 `mock` 模式，即使没有真实 AI Key 也能演示功能。当前国内体验服务器已切换为 DeepSeek 真实模型，右侧 AI 状态卡会显示 `真实模型`、provider 和 model。

如果要在本地启用 DeepSeek：

1. 复制环境变量文件：

```bash
cp backend/.env.example backend/.env
```

2. 修改 `backend/.env`：

```env
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=你的_API_Key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

3. 重启后端。

如果要切回无 Key 演示模式，把 `AI_PROVIDER` 改回 `mock`。

## 3. 页面总览

首次打开页面会先进入登录/注册页。登录后进入学习工作台。

```text
左侧导航            中间主工作区                         右侧复盘区
────────────────────────────────────────────────────────────────────
写作台              输入、翻译、润色、推荐语料              累计表达
表达库              已收藏表达                             待改进表达
表达改进库          已记录改进建议                          今日目标

```

## 4. 写作台使用说明

写作台是最主要的工作区。

手机端会自动切换为单栏布局，底部标签栏用于切换写作台、表达库、表达改进库、历史详情、学习档案和社群互学。右侧桌面复盘栏在手机端会收起，避免页面过长和布局变形。

### 4.1 输入原文

在“原文输入”框中输入你想翻译或润色的内容。

示例：

```text
今天我想练习一段关于校园学习生活的英文表达。
```

也可以输入英文句子用于润色：

```text
I want practice English writing today.
```

### 4.2 填写语境说明

在“语境说明”中填写使用场景。

示例：

- 课程作业
- 正式邮件
- 校园生活分享
- 学术论文摘要
- 留学申请文书

语境说明会传给后端，让 AI 输出更贴近场景。

### 4.3 选择目标语言

当前可选：

- English
- Japanese
- Korean
- Chinese

### 4.4 快速模式和深度模式

| 模式 | 适合场景 | 输出特点 |
|---|---|---|
| 快速 | 查词、查短句、快速理解 | 速度优先，返回直接译文和简短建议 |
| 深度 | 作业、正式表达、希望学习修改思路 | 初译 + 审校，返回译文、审校说明和学习建议 |

注意：

- “快速/深度”只影响“翻译”按钮。
- “快速/深度”按钮本身不会立即调用 AI，需要点击“翻译”后生效。
- “润色”按钮目前固定走润色流程，不受快速/深度切换影响。
- 深度模式会调用两次模型，速度更慢，费用通常也高于快速模式，但更适合正式文本和学习复盘。

### 4.5 点击“翻译”

操作：

1. 输入文本。
2. 选择目标语言。
3. 选择快速或深度。
4. 点击“翻译”。

结果区会显示：

- 翻译类型。
- provider/model。
- 译文。
- 审校说明。
- 学习建议。

翻译完成后：

- 自动新增一条历史记录。
- 今日目标自动 +1。
- 累计表达自动 +1。

### 4.6 点击“润色”

操作：

1. 输入外语文本。
2. 填写语境说明。
3. 点击“润色”。

结果区会显示：

- 润色版本。
- 修改说明。
- provider/model。

润色完成后：

- 自动新增历史记录。
- 今日目标自动 +1。
- 可继续收藏表达或加入改进记录。

### 4.7 语音输入

点击“中文语音”按钮后，浏览器会请求麦克风权限。

使用说明：

- 当前轻量版仅支持中文语音输入。
- 浏览器支持时，会显示“识别中”的即时转写反馈。
- 每次语音输入最多 10 秒，最多写入 50 字。
- 识别到的中文文本会追加到原文输入框。
- 如果浏览器不支持，会显示提示。
- 建议在 Chrome 或支持 Web Speech API 的浏览器中使用。

注意：

- 语音识别依赖浏览器能力。
- 本地开发环境通常可用，线上环境可能需要 HTTPS。
- 如果需要更稳定的正式线上语音能力，后续可接入云端语音识别服务。

### 4.8 沉浸模式

点击“沉浸”按钮后：

- 输入区域变大。
- 文本字号变大。
- 更适合长段写作。

再次点击“退出沉浸”可恢复普通模式。

## 5. AI 输出区操作

### 5.1 收藏表达

以下内容可以收藏到表达库：

- 翻译结果。
- 润色版本。
- 翻译建议。
- 推荐语料。
- 历史详情中的译文或终稿。

点击“收藏表达”后，可在左侧“表达库”中查看。

### 5.2 加入改进记录

以下内容可以加入表达改进库：

- 深度审校说明。
- 润色修改说明。
- 历史详情中的审校/修改建议。

点击“加入改进记录”后，可在左侧“表达改进库”中查看。

### 5.3 分享社群

译文和润色版本可以点击“分享社群”。

分享后：

- 自动切换到“社群互学”页面。
- 内容会保存在本地社群列表。

当前社群是本地演示版，不会上传到服务器。

## 6. 推荐语料

写作台下方有“推荐语料”面板。

系统会根据：

- 原文输入。
- 最近翻译结果。
- 最近润色结果。

匹配内置语料库中的表达。

示例语料：

| 主题 | 表达 |
|---|---|
| Campus Life | campus learning and daily life |
| Academic Writing | develop a more natural academic expression |
| Translation Review | revise the translation for accuracy and fluency |
| Language Practice | build a sustainable language learning habit |

你可以点击“收藏表达”把语料加入表达库。

## 7. 表达库

进入方式：

左侧导航 → 表达库

表达库保存你认为值得复习和模仿的表达。

每条表达包含：

- 来源。
- 表达文本。
- 移除按钮。

常见来源：

- 译文。
- 翻译建议。
- 推荐语料。
- 历史译文。
- 润色版本。

## 8. 表达改进库

进入方式：

左侧导航 → 表达改进库

表达改进库保存你需要复盘的表达优化建议、修改理由和审校说明。

每条改进记录包含：

- 来源。
- 修改说明。
- 移除按钮。

建议使用方法：

1. 每次润色后，把你不熟悉的修改加入表达改进库。
2. 每天复习表达改进库。
3. 尝试用自己的话重新造句。

## 9. 历史详情

进入方式：

- 点击右侧“最近记录”中的任意一条。
- 或点击左侧“历史详情”。

历史详情展示：

- 初稿。
- 译文或终稿。
- 审校说明。
- 学习建议。
- 修改说明。

你可以在历史详情中继续：

- 收藏表达。
- 加入改进记录。
- 回到写作台继续修改。

说明：

- 右侧只显示最近 6 条。
- 系统会完整保存历史记录，不只保存 6 条。

## 10. 学习档案

进入方式：

左侧导航 → 学习档案

学习档案包括：

### 10.1 顶部统计

- 历史记录。
- 表达收藏。
- 表达改进。

### 10.2 每日目标

默认目标为每天 3 条学习产出。

你可以修改“每日条目”数字。

今日目标的计数来源：

- 今日完成的翻译。
- 今日完成的润色。
- 今日收藏的表达。
- 今日加入的改进记录。

### 10.3 连续天数

如果今天和前几天都有学习产出，会显示连续学习天数。

### 10.4 近 7 天练习趋势

柱状图展示最近 7 天每天的学习产出数量。

### 10.5 14 天热力

热力格展示最近 14 天的学习活跃度。

颜色越深，说明当天学习记录越多。

## 11. 社群互学

进入方式：

左侧导航 → 社群互学

社群互学当前是本地初版，用于演示共享语料和同伴启发。

可分享：

- 翻译结果。
- 润色版本。

每条共享内容包含：

- 来源。
- 时间。
- 文本。
- 移除按钮。

## 12. 右侧今日复盘

右侧固定显示：

### 12.1 累计表达

统计口径：

```text
累计表达 = 完整历史记录数 + 表达库数量
```

### 12.2 待改进表达

包括：

- 表达改进库数量。
- 当前结果中的审校说明。
- 当前润色结果中的修改说明。

### 12.3 今日目标

显示：

- 今日已完成数量。
- 每日目标数量。
- 进度条。

### 12.4 最近记录

展示最近 6 条历史。

点击任意记录可进入历史详情。

点击“清空”会清空历史记录。

### 12.5 演示工具

右侧底部提供一组演示工具，主要用于同学试用、课堂展示和答辩前准备。

#### 导出报告

点击“导出报告”后，系统会生成一份 Markdown 格式学习报告。

报告内容包括：

- 导出时间。
- 历史记录数量。
- 表达收藏数量。
- 表达改进数量。
- 社群分享数量。
- 今日目标完成情况。
- 最近练习。
- 最近收藏表达。
- 最近改进记录。
- 复盘建议。

文件名格式类似：

```text
zjsuer-learning-report-2026-04-30.md
```

这份报告适合用于：

- 阶段性学习复盘。
- 课程作业材料整理。
- 大创答辩时展示“可复盘学习闭环”。

#### 载入演示数据

点击“载入演示数据”后，系统会写入一组模拟学习记录，包括：

- 深度翻译记录。
- 润色记录。
- 快速翻译记录。
- 表达库条目。
- 表达改进库条目。
- 社群分享内容。
- 每日目标配置。

载入后会自动跳转到学习档案，方便立刻查看趋势图、热力格、累计表达和目标进度。

该功能适合：

- 第一次打开项目时快速理解完整流程。
- 给没有真实使用数据的同学演示产品。
- 答辩前快速恢复一个“有内容”的展示状态。

#### 清空全部数据

点击“清空全部数据”后，会清除当前浏览器里的学习数据，包括：

- 历史记录。
- 表达库。
- 表达改进库。
- 社群分享。
- 每日目标配置。
- 当前 AI 输出结果。

清空后页面会回到写作台。

注意：该操作只清空当前浏览器本地数据，不会影响 GitHub 仓库代码。

## 13. 数据保存说明

当前学习数据会同时保存在浏览器本地缓存和后端个人数据空间中。

保存内容：

| 数据 | 保存位置 |
|---|---|
| 登录账号 | 后端 SQLite |
| 历史记录 | localStorage + 后端 SQLite |
| 表达库 | localStorage + 后端 SQLite |
| 表达改进库 | localStorage + 后端 SQLite |
| 每日目标 | localStorage + 后端 SQLite |
| 社群共享 | localStorage + 后端 SQLite |

注意：

- 登录同一账号后，会从后端加载个人学习数据。
- 浏览器 localStorage 仍作为本机缓存，便于弱网或刷新时恢复状态。
- 清空全部数据会清空当前账号下的学习记录。
- 当前版本不会把学习内容上传到云端。

## 14. 常见问题

### 14.1 为什么显示的是 placeholder？

因为后端处于 `mock` 模式。要使用真实模型，请配置 DeepSeek、OpenAI、DashScope 或其他兼容 Provider 的 API Key。当前腾讯云国内体验环境已切换为 DeepSeek，正常情况下不应再显示 placeholder。

### 14.2 为什么语音按钮不可用？

可能原因：

- 浏览器不支持 Web Speech API。
- 未授予麦克风权限。
- 当前环境不支持语音识别。
- 公网 HTTP 地址可能无法稳定使用麦克风或语音识别能力。
- 说话时间超过 10 秒或识别文字超过 50 字，系统会自动停止或截断。

解决方式：

- 使用 Chrome。
- 检查麦克风权限。
- 继续使用文本输入。
- 后续配置 HTTPS 后再把语音输入作为正式展示功能重点测试。

### 14.3 为什么今日目标会自动增加？

完成一次翻译或润色就算一次学习产出，因此会自动 +1。收藏表达和加入改进记录属于额外学习沉淀，也会计入今日学习活动。

### 14.4 为什么右侧只有 6 条记录？

右侧是最近记录区域，只展示最近 6 条。完整历史仍然保存在本地，用于学习档案和统计。

### 14.5 为什么打开学习档案时稍微慢一点？

学习档案使用 ECharts 图表库。为了减少首页加载体积，图表库采用动态加载，只有进入学习档案时才加载。

## 15. 推荐使用流程

### 15.1 日常写作训练

1. 打开写作台。
2. 输入原文。
3. 填写语境说明。
4. 使用深度翻译。
5. 阅读审校说明。
6. 收藏优秀表达。
7. 将不懂的修改加入改进记录。
8. 查看历史详情复盘。

### 15.2 外语句子润色

1. 输入外语句子。
2. 点击润色。
3. 对照润色版本。
4. 阅读修改说明。
5. 把修改说明加入表达改进库。
6. 每天复习表达改进库。

### 15.3 备赛或课程展示

1. 准备一段中文校园生活文本。
2. 使用快速模式展示即时翻译。
3. 切换深度模式展示初译+审校理念。
4. 使用润色展示修改说明。
5. 收藏表达和改进记录。
6. 展示历史详情。
7. 展示学习档案趋势图和热力格。
8. 展示社群互学。
